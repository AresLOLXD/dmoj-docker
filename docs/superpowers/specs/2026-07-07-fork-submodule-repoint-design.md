# Point dmoj/repo submodule at AresLOLXD/online-judge fork

Date: 2026-07-07

## Context

`dmoj/repo` (the DMOJ Django app submodule) currently tracks upstream
`DMOJ/online-judge`, pinned at `3b3ecec8` (`v4.0.0-192-g3b3ecec8`). The user
maintains a fork, `AresLOLXD/online-judge`, whose `master` branch (`cd43999e`)
is a fast-forward descendant of upstream's current `master` (`6aaddea6`) plus
40 additional commits (Cloudflare Turnstile registration protection,
submission rate limiting, admin muting/banning actions, extra API metadata,
etc.). No upstream sync is needed — the fork is already ahead of both the
currently-pinned commit and upstream's tip.

This repoint makes the fork's `master` the tracked branch going forward, so
site-specific improvements live in the submodule the same way judge
improvements already live in the `AresLOLXD/judge-server` fork used by
`judge-tier3-1` (see
`docs/superpowers/specs/2026-07-02-judge-addkarel-integration-design.md`).

This spec was scoped out of that earlier judge-integration design as a
"freshness check of dmoj/repo against upstream" follow-up; it is now
superseded by switching to the fork entirely rather than tracking upstream.

## Change

1. **`.gitmodules`**: change `url` to `https://github.com/AresLOLXD/online-judge`
   and add `branch = master`, so `git submodule update --remote` (used by
   `scripts/update`) tracks the fork's `master` rather than upstream's.
2. **`dmoj/repo` submodule**:
   - `git remote set-url origin https://github.com/AresLOLXD/online-judge`
   - `git fetch origin`
   - `git checkout master` (lands on `cd43999e`)
   - Untracked local files inside the submodule (`uwsgi.ini`,
     `websocket/config.js` — generated by `scripts/initialize`) are not
     touched by this switch; they aren't tracked by git.
3. **Outer repo commit**: stage `.gitmodules` and the updated submodule
   pointer (`dmoj/repo` now at `cd43999e`), commit together.
4. **Documentation updates** — replace references to the submodule tracking
   upstream `DMOJ/online-judge` with the fork, in:
   - `CLAUDE.md` (repo-layout section: "the actual DMOJ site source code
     lives in the `dmoj/repo` git submodule (`DMOJ/online-judge`)")
   - `CLAUDE.local.md` (same wording, private copy)
   - `README.md` (intro line linking to `DMOJ/online-judge`)

## Out of scope

- Any code changes inside the submodule itself — this only changes which
  remote/branch it points to.
- The security review of the Docker setup (separate investigation, to be
  done after this change using `/security-review`).
- Re-running `scripts/copy_static` / migrations — the fork's extra commits'
  operational impact (e.g. new Turnstile settings) is handled as a follow-up
  once the repoint lands, not part of this spec.

## Testing / verification

- `git submodule status` shows `dmoj/repo` at `cd43999e` with no `+`/`-`
  prefix (not diverged, not out of sync with `.gitmodules`).
- `cd dmoj/repo && git remote -v` shows the fork URL.
- `git log -1 dmoj/repo` (from outer repo) shows the fork's latest commit
  message ("Add opt-out coverage test and key-asymmetry deployment note").
- `grep -r "DMOJ/online-judge" CLAUDE.md CLAUDE.local.md README.md` returns
  no matches (or only matches that are intentionally left, e.g. a changelog
  mentioning the original upstream project).
